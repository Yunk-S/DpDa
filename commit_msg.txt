Silence chart_generator non-fatal errors at startup

Chart generation runs at app import time to verify all expected
charts exist. Old LightGBM models trained with mismatched categorical
features fail predict() on newer LightGBM versions. These failures
are non-fatal: charts already exist on disk, server keeps working.

Replace verbose ERROR + traceback logs with concise WARNING lines
to reduce noise during startup. The app already wraps this call in
a try/except so the server starts regardless.
